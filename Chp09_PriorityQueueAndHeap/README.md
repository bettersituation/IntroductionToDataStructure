# Chapter9. 우선순위 큐와 힙


#### 우선순위 큐

- 우선순위 큐란 큐와 유사하지만 dequeue 를 하면 삽입된 순서에 상관없이 가장 우선순위가 높은 데이터가 나오는 큐이다.
- 우선순위 큐를 구현하는 방법에는 여러가지가 있지만 힙을 사용하는 것이 효율적이어서 주로 힙으로 구현한다.


#### 힙

- 힙은 Complete binary tree, 즉 완전 이진 트리이고 추가적으로 **모든 노드는 자식 노드보다 값이 커야 한다** 는 제약조건을 가진다.
- 그러므로 루트 노드의 값이 제일 클 것이다.
- 이러한 힙의 특성 때문에 힙을 이용하여 우선순위 큐를 구현하는 것이 일반적이다.
- 삽입과 삭제의 시간 복잡도는 O(log n) 이다.
- 삽입 및 삭제를 간편하게 구현하기 위해 힙을 구현할 때 배열을 사용하는 것이 일반적이다.
- 삽입과 삭제를 구현할 때 우선순위에 따라 해당 데이터의 위치가 변경되어야 하므로 재귀적으로 구현되는 것이 일반적이다.
- 루트 노드의 번호를 1, 왼쪽 자식 노드를 2, 오른쪽 자식 노드를 3, 왼쪽 자식 노드의 왼쪽 자식 노드를 4, ... 이라고 했을 때 아래 성질을 만족한다.
- 1) 현재 노드의 번호를 n 이라고 하면 왼쪽 자식 노드의 번호는 2\*n, 오른쪽 자식 노드의 번호는 2\*n + 1 이다.
- 2) 현재 노드의 번호를 n 이라고 하면 부모 노드의 번호는 n/2 이다.
