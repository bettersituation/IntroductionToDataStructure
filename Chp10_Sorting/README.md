# Chapter10. 정렬

배열 자료구조의 정렬에 대해 다룬다.  

#### 1. 버블 정렬

- 인접한 두 원소를 비교하여 정렬하는 것을 마지막 원소까지 수행한다.
- 예를 들어 아래 같은 배열은 다음과 같은 과정을 거쳐 정렬된다


|step|data1|data2|data3|data4|  
|---|---|---|---|---|
|0|**3**|2|4|1|
|1|2|**3**|4|1|
|2|2|3|**4**|1|
|3|2|3|1|**4**(O)|
|4|**2**|3|1|4(O)|
|5|2|**3**|1|4(O)|
|6|2|1|**3**(O)|4(O)|
|7|**2**|1|3(O)|4(O)|
|8|1(O)|**2**(O)|3(O)|4(O)|

- 구현은 쉬우나 시간복잡도 O(n^2) 으로 성능이 좋은 편은 아니다.


#### 2. 선택 정렬

- 버블 정렬과 비슷하고 아래 과정을 거쳐 정렬된다.  
- 전체 배열에서 제일 처음 제일 작은 값은 찾아 인덱스 0 에 삽입한다.
- 인덱스 1 부터 시작하는 부분 배열에서 제일 작은 값을 찾아 인덱스 1 에 삽입한다.
- 인덱스 2 부터 시작하는 부분 배열에서 제일 작은 값을 찾아 인덱스 2 에 삽입한다.
- ...반복...
- 버블 정렬과 마찬가지로 구현은 쉬우나 시간복잡도 O(n^2) 으로 성능이 좋은 편은 아니다.


#### 3. 삽입 정렬

- 배열 중 앞부분 일부가 정렬되어 있을 때 쓰면 좋은 방법이다. 아래와 같은 과정을 거쳐 정렬한다.  

|step|data1|data2|data3|data4|data5|
|---|---|---|---|---|---|
|0|5|3|2|4|1|
|1|3(O)|5(O)|2|4|1|
|2|3(O)|5(O)|__|4|1|
|3|3(O)|__|5(O)|4|1|
|4|2(O)|3(O)|5(O)|4|1|
|5|2(O)|3(O)|5(O)|__|1|
|6|2(O)|3(O)|4|5(O)|1|

...

- 버블 정렬, 삽입 정렬과 마찬가지로 시간 복잡도가 O(n^2) 이다.


#### 4. 힙 정렬

- 힙에 삽입하고 빼옴으로써 정렬하는 방법이다.
- 하나의 데이터를 삽입 및 추출하는 데에 O(log n) 의 시간복잡도가 소요되고, n 번 수행해야 하므로 최종적인 시간복잡도는 O(n log n) 이다.
