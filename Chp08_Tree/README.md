# Chapter8. 트리


#### 트리 용어
- 노드(Node)
- 간선(Edge)
- 루트 노드(Root Node)
- 단말 노드(Termibal Node) a.k.a 잎사귀 노드(Leaf Node)
- 내부 노드(Internal Node) a.k.a 비단말 노드(Nonterminal Node)
- 부모(Parent)
- 자식(Child)
- 형제(Siblings)
- 조상(Ancestor)
- 후손(Descendant)
- 레벨
- 높이


#### 이진 트리(Binary Tree) 의 정의
- 조건1. 이진 트리는 루트 노드를 중심으로 두 개의 서브 트리로 나뉜다.  
- 조건2. 나뉘어진 두 서브 트리 역시 이진 트리여야 한다.  


#### 포화 이진 트리(Full Binary Tree)
- 포화 이진 트리란 주어진 높이 안에서 꽉 차있어서 추가적인 데이터를 삽입하기 위해서는 높이가 높아져야 하는 이진 트리이다.


#### 완전 이진 트리(Complete Binary Tree)
- 완전 이진 트리란 노드가 위에서 아래로, 왼쪽에서 오른쪽으로 차례대로 꽉 채워져 있는 트리이다. 포화 이진 트리에서 몇 개의 데이터를 추가적인 레벨에서 왼쪽부터 삽입한다면 이는 완전 이진 트리가 될 것이다.  


#### 트리의 구현
- 배열이나 연결 리스트를 사용하여 구현 가능하다. 보통 편의상 연결 리스트를 사용하는데 힙 등의 자료구조에서는 배열을 사용한다.  
- 배열을 사용할 때는 구현 편의성을 위해 맨 처음 한 개 값을 비워둔다.  


#### 이진 트리 순회
- 전위 순회(Preorder Traversal): 루트 노드 순회 &rarr; 왼쪽 노드 순회 &rarr; 오른쪽 노드 순회
- 중위 순회(Inorder Traversal): 왼쪽 노드 순회 &rarr; 루트 노드 순회 &rarr; 오른쪽 노드 순회
- 후위 순회(Postorder Traversal): 왼쪽 노드 순회  &rarr; 오른쪽 노드 순회 &rarr; 루트 노드 순회
- 순회는 재귀적으로 수행된다.


#### 수식 트리
- 수식 트리는 중위 표현식 &rarr; 후위 표현식 &rarr; 수식 트리 변환의 과정을 거쳐 생성된다.  
- 수식 트리를 전위 순회하면 전위 표현식, 중위 순회하면 중위 표현식, 후위 순회하면 후위 표현식으로 나타난다.  


#### 느낀 점

트리는 그 자료구조의 특성 상 재귀적으로 정의되기 때문에 디버깅 과정이 아무래도 선형 자료구조보다 힘든 듯하다.  
또한 이 챕터를 위해 다른 챕터보다 많은 파일을 만들었는데 약간 혼동이 왔다.  
실제로 트리 구조를 쓰지 말자고 할 수도 없고 파일을 적게 만들수도 없기 때문에 결국 이런 혼동을 줄이기 위해서는  
명확한 프로젝트 구조가 필요하구나 하는 새삼스러운 생각을 잠시 해보았다..
